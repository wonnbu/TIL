### 1. 오늘 배운 내용
- 상속과 추상화
    - 추상 클래스(Role) : 공통 속성을 관리. 하지만 Role 자체로는 완벽한 존재가 아니기에 getInfo()와 getPolicy()를 추상 메서드로 두어 자식들이 반드시 구현하게 함
    - 자식 클래스(Lion, Staff) :  Role을 상속받아 각각 studentNumber(학번)와 position(직책)이라는 자신만의 특징을 추가
- 인터페이스 활용
    - 인터페이스(Policy) : "과제를 제출할 수 있는가?"라는 규칙의 틀(submit())을 정의
    - LionPolicy, StaffPolicy :  이 틀을 실제로 구현하여 아기사자는 true, 운영진은 false 반환
- 다형성
    - 타입의 통합 : Lion 객체와 Staff 객체를 모두 Role 이라는 부모 타입의 배열에 담음
    - 동적 바인딩 : role.getInfo()를 호출할 때, Java는 실행 시점에 이 객체가 Lion 인지 Staff 인지 판단하여 각각의 클래스에 오버라이딩된 메서드를 알아서 호출
    - 위임 : role.getPolicy().submit() 부분에서 Role 대신 자신에게 할당된 Policy 객체에게 제출 가능 여부 판단 넘김

### 2. 핵심 정리(나의 언어로)
같은 변수 하나로 관리하려면 공통 부모 클래스 만들고 각각 상속하기. extends로 상속했으면 super()로 부모클래스 생성자 호출. 즉, 공통 변수를 호출하고 this생성자로는 자식클래스의 자기 고유 정보만 처리하면 됨.
인터페이스(이 메서드 있어야 함) vs 클래스(이 메서드 이렇게 동작함) / 오버라이딩(@Override)은 부모 메서드 재정의
private은 해당 클래스 안에서만 사용 가능하지만(getter필요), protected는 자식 클래스까지 사용 가능(this생성자 먹힌다는 뜻)
if문 안쓰기 위해서 향상된 for문 사용 가능. 배열을 통해서 아기사자와 운영진의 정보를 순서대로 출력 가능
